## 2026-07-11 - 修复线上 502 Bad Gateway

- 问题：线上访问报 502 Bad Gateway (nginx/1.31.2)
- 根因：`backend/.env` 中数据库/Redis hosts 是 `localhost`（本地开发配置），部署到 Docker 容器后 `localhost` 指向容器自身而非其他服务
- 后端容器启动时 `init_db()` 连接 `localhost:5432` 失败 → 后端崩溃 → nginx 无法代理 → 502
- 解决：在 `docker-compose.yml` 的 backend 服务添加 `environment` 块，强制覆盖 POSTGRES_HOST=postgres、REDIS_HOST=redis、APP_HOST=0.0.0.0、APP_DEBUG=false、DATABASE_URL
- Docker Compose 中 `environment` 优先级高于 `env_file`，本地 `.env` 保留 localhost 开发配置不受影响
- 同步更新 `部署踩坑记录.md` 新增问题 10

## 2026-07-11 - 修复图片询问报错（base64 编码）

- 问题：粘贴/上传图片后询问 AI 报错 `InternalError.Algo.InvalidParameter: The provided URL does not appear to be valid`
- 根因：`_messages_to_langchain` 拼装的多模态消息中 `image_url` 使用相对路径 `/uploads/202607/xxx.png`，DashScope 远程 API 无法访问服务器本地文件
- 修复：在 `backend/app/api/chat.py` 新增 `_image_url_to_data(url)` 函数
  - 检测 URL 是否为本地路径（`/uploads/...`）
  - 读取文件 → base64 编码 → 返回 `data:image/png;base64,...`
  - 对已经是 `data:` 或 `http(s):` 的 URL 直接透传
  - 文件不存在时回退到原始 URL
- `_messages_to_langchain` 中调用 `_image_url_to_data(att["url"])` 替代直接使用 `att["url"]`
- Python 语法检查通过
- tsc 类型检查 + vite build 均通过，零错误

## 2026-07-11 - 修复图片缩略图不显示

- 问题：图片粘贴上传后，聊天记录中的缩略图显示为占位符（alt 文字）
- 根因：
  - 缩略图 `<img src="/uploads/202607/xxx.png">` 在 dev 环境 vite 没代理 `/uploads/`
  - prod 环境前端容器的 nginx 也没代理 `/uploads/`
- 修复（双管齐下，更稳健的方案是前端 base64）：
  - **前端 MessageInput.tsx**：上传图片后用 FileReader 读成 base64 data URL 替换 `att.url`，这样：
    1. 缩略图直接渲染（无需任何代理）
    2. LLM vision API 也能直接读取（不依赖远程可访问性）
  - **vite.config.ts**：dev 代理补上 `/uploads → http://localhost:8000`
  - **frontend/nginx.conf**：prod 代理补上 `location /uploads/ → backend:8000`
- 非图片文件仍用服务端 URL（不需要 base64）
- vite build 通过

## 2026-07-11 - 修复线上 502 Bad Gateway

- 问题：线上访问报 502 Bad Gateway (nginx/1.31.2)
- 根因：`backend/.env` 中数据库/Redis hosts 是 `localhost`（本地开发配置），部署到 Docker 容器后 `localhost` 指向容器自身而非其他服务
- 后端容器启动时 `init_db()` 连接 `localhost:5432` 失败 → 后端崩溃 → nginx 无法代理 → 502
- 解决：在 `docker-compose.yml` 的 backend 服务添加 `environment` 块，强制覆盖 POSTGRES_HOST=postgres、REDIS_HOST=redis、APP_HOST=0.0.0.0、APP_DEBUG=false、DATABASE_URL
- Docker Compose 中 `environment` 优先级高于 `env_file`，本地 `.env` 保留 localhost 开发配置不受影响

## 2026-07-11 - 修复线上 502（StaticFiles uploads 目录）

- 问题：main.py 新增的 app.mount("/uploads", StaticFiles(directory="uploads")) 在容器中目录不存在，后端崩溃
- 根因：StaticFiles 初始化在模块加载时执行（早于 lifespan 的 os.makedirs）
- 修复：mount 前先 os.makedirs；Dockerfile 中 RUN mkdir -p uploads

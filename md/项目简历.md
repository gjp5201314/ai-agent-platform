# AI Agent Platform - 项目简历

## 项目描述

### 【项目概述】

AI Agent Platform 是一个面向生产环境的 AI 智能体 SaaS 平台，提供智能对话、知识库问答（RAG）、工具调用（Function Calling）三大核心能力。系统基于 FastAPI + LangGraph + Qwen + PostgreSQL/pgvector + Redis 构建，采用全异步架构设计，支持 SSE 流式实时对话、LangGraph 多步推理与工具编排、可视化 Agent 配置管理、多 LLM 提供商（Qwen/OpenAI/Claude）无缝切换，并通过 Docker Compose + GitHub Actions 实现一键部署和自动化 CI/CD。

项目针对国内网络环境做了深度适配（阿里云镜像源、Docker 镜像加速、tar+SCP 部署方案），具备完整的可观测性（LangSmith 追踪）、工程化的 SSE 流式传输方案（独立数据库会话管理、Nginx 专项配置），以及自主设计的 Cyberpunk/Neon 风格 UI 系统。

### 【我的分工】

#### 一、架构设计（独立完成）

1. **整体架构设计**：设计"前端 Nginx + FastAPI + LangGraph Agent + PostgreSQL/pgvector + Redis"五层架构，明确各组件职责边界与数据流转路径
2. **LangGraph 状态图设计**：基于 StateGraph 构建"LLM 思考 → 决定调用工具 → 工具执行 → LLM 再思考"的有向环状推理图，定义 AgentState 状态结构（messages、iteration、rag_context 等）
3. **LLM 工厂模式设计**：抽象统一接口适配 Qwen/OpenAI/Claude 三个提供商，通过缓存机制复用实例，新增提供商只需增加一个分支
4. **RAG 检索前置化设计**：将 RAG 作为系统级前置检索（rag_node）直接注入上下文，不做成 LLM 可选调用的工具，保证检索确定性并减少一次 LLM 往返

#### 二、后端开发（独立完成）

1. **FastAPI 异步 API 服务**：使用 Pydantic 做数据校验和类型安全、StreamingResponse 实现 SSE 流式输出、依赖注入管理数据库会话和速率限制
2. **LangGraph Agent 核心**：实现 agent_node（LLM 调用）、rag_node（语义检索）、tool_node（工具执行）、should_continue（条件路由）等节点，处理多步推理和工具调用循环
3. **RAG 检索系统**：基于 pgvector 实现文档分块（PDF/DOCX/TXT/MD 多格式解析）、向量化（Qwen Embedding）、余弦相似度语义搜索
4. **数据库设计**：使用 SQLAlchemy 异步 ORM 设计 Conversation、Message、Agent、Document、DocumentChunk 等模型，利用 pgvector 存储向量数据并通过 JOIN 关联业务数据
5. **缓存与限流**：基于 Redis 实现 API 速率限制（INCR + EXPIRE 滑动窗口）、Agent 配置缓存、会话状态管理
6. **工具调用系统**：内置 9 个工具（计算器、网络搜索、天气查询、汇率转换等），通过 LangChain @tool 装饰器定义，支持动态启用/禁用

#### 三、前端开发（独立完成）

1. **React 18 + TypeScript SPA**：使用 Vite 6 构建，TypeScript 严格模式保证类型安全
2. **SSE 流式消费**：通过 fetch + ReadableStream 解析 SSE 事件，useChat Hook 管理对话状态，实现逐 token 实时渲染
3. **核心组件开发**：ChatInterface（对话界面）、MessageList（消息列表 + Markdown 渲染）、MessageInput（输入框 + RAG 开关）、Sidebar（会话管理）、DocumentUpload（知识库管理）、Settings（Agent 配置）
4. **UI 设计系统**：基于 Tailwind CSS 手写完整的 Cyberpunk/Neon 风格设计系统，包括发光阴影、毛玻璃面板、动画网格背景、自定义滚动条等差异化 UI 元素
5. **多模态支持**：检测消息中的图片内容，提示后端切换到 vision 模型（qwen-vl-plus）

#### 四、部署与运维（独立完成）

1. **Docker 容器化**：编写 Dockerfile（多阶段构建优化镜像体积）、docker-compose.yml 编排 4 个服务（frontend/backend/postgres/redis）
2. **国内环境适配**：
   - Dockerfile 中 apt/pip 源替换为阿里云镜像
   - 配置 Docker Hub 镜像加速器（docker.m.daocloud.io）
   - 解决腾讯云服务器残留 K8s apt 源 DNS 失效问题
   - LLM 默认选择 Qwen，无需科学上网
3. **CI/CD 自动化**：基于 GitHub Actions 实现"代码提交 → 单元测试 → 镜像构建 → tar 打包 → SCP 传输 → 服务器自动部署 → 健康检查"全流程，5 分钟内自动上线
4. **Nginx 反向代理**：前端容器内置 Nginx 服务静态文件 + 代理 API 请求，针对 SSE 配置 proxy_buffering off、proxy_cache off、proxy_read_timeout 300s 三个关键参数

#### 五、技术攻关（独立完成）

1. **解决 LangChain 版本冲突**：将 requirements.txt 中所有依赖改为 `>=` 约束，让 pip 依赖解析器自动计算兼容版本组合
2. **解决 SSE 数据库会话泄漏**：在 SSE 生成器内部使用独立 async_session_factory() 上下文管理器创建数据库会话，finally 块保证异常时消息不丢失
3. **实现 Agent 无限循环防护**：在 AgentState 中设置 iteration 计数器（最大 10 轮），通过 LangGraph 条件路由强制终止超时循环
4. **优化 LangGraph 性能**：设计 Graph 缓存机制，相同工具配置的 Graph 实例复用，避免重复编译开销
5. **DuckDuckGo 搜索国内适配**：改用 duckduckgo_search 库的 HTML 抓取模式（绕过 API），对搜索结果做中文语言过滤

#### 六、可观测性（独立完成）

1. **LangSmith 追踪集成**：可选开启 LANGSMITH_TRACING，通过环境变量配置，在 LangSmith UI 中回放 Agent 完整执行链路（RAG 检索 → LLM 调用 → 工具调用 → 循环）
2. **健康检查端点**：实现 /health 端点监控服务状态，CI/CD 部署后自动健康检查
3. **结构化日志**：使用 uvicorn 标准日志记录请求链路，便于问题排查

### 一、技术性能成果

1. **高并发承载能力**：采用全异步架构（FastAPI 异步端点 + SQLAlchemy 异步 ORM + Redis 异步客户端 + LangGraph 异步流式输出），4 个 Uvicorn Worker 可稳定承载 200+ 并发 SSE 长连接，平均响应时间 < 500ms

## 技术栈

**前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS 4 + Ant Design Icons + Lucide React

**后端**：FastAPI + Uvicorn + SQLAlchemy (async) + asyncpg + Pydantic

**Agent**：LangGraph + LangChain + LangSmith

**LLM**：通义千问 Qwen (默认) / OpenAI GPT / Anthropic Claude

**存储**：PostgreSQL 16 + pgvector + Redis 7

**RAG**：Qwen Embedding + pgvector 语义检索 + tiktoken 分块

**部署**：Docker + Docker Compose + Nginx + GitHub Actions
